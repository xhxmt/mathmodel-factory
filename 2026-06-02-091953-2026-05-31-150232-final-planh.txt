  先把"准确度"拆成三层（加模型只对其中两层有效）

  1. 方差（随机误差）：同一篇跑多次分数飘（haiku 的 13–16 分 spread）。← 采样 K
  + 多模型集成能压。
  2. 偏置（系统误差）：系统性高估某维、顶端给分偏松（haiku 给到 99.3）。←
  加不同家族模型能平均掉各自的特异偏置；但若几个小模型有共同盲区（都不擅长数学严
  谨性），平均出来是"更稳的错"，不是"更准"。
  3. 能力天花板：能不能把 78 分和 85 分的论文区分开。←
  只有模型能力本身决定，加同档模型修不动。

  多模型面板（PoLL）确实有效，证据明确

  《Replacing Judges with Juries》 (https://arxiv.org/abs/2404.18796) 发现：3
  个不同家族的小模型组成面板，与人类判断的相关性优于单个 GPT-4，便宜约
  7×，且因家族异质显著降低自我偏好/家族偏置。机制正是上面第 1、2
  层——跨家族抵消各自偏置，而单个大模型反而会"过度推理"。

  这对我们正好对症：haiku + Gemini 3.5 Flash + DeepSeek V4 Flash = 标准的三家族
  PoLL 配方，能
  - 压 haiku 方差（跨家族中位数 > haiku 自己重跑 K 次）；
  - 强独立性：我们用 gpt-5.5/codex 写论文，评委用 Claude+Google+DeepSeek
  三家全非 OpenAI 系；
  - 把 haiku 顶端宽松给分被更严的同伴拉回；
  - 跨模型分歧度本身成为信号（三家分歧大 = 这篇有争议、分数不可信）。

  而且 2024 年 PoLL 的"小模型"（command-r/gpt-3.5/haiku）和 2026 的 flash
  不是一回事：Gemini 3.5 Flash (https://techcrunch.com/2026/05/19/with-gemini-3-
  5-flash-google-bets-its-next-ai-wave-on-agents-not-chatbots/) 推理已接近

  1. 没有专家标注样本，你测不出"更准"。 PoLL
  能证明面板更好，是因为它有人类判断做基准。我们现在 n=2、零 ground truth——加模
  型只会提升"模型之间的一致性"，而几个模型可以一致地错。这是第一优先级：攒 10–20

  1. 先有一小撮带标注的验证集 —— 否则一切都是盲优化。

  真实有效、便宜、有论文背书；但要做成跨家族而非堆同档重复，聚合用中位数 +

  只有"优秀"意味着什么（诚实）

  全是高分样本 = 退化的验证集，两个直接后果：

  1. 测不了区分度（range 
  restriction）。所有样本标签都挤在"优"，你算不出有意义的"评委分 vs
  专家分"相关性——没有低端做对比，Kendall τ / Cohen κ
  这类指标在受限值域上会失真。
  2. 正好对我们记录的失败模式失明。我们 baseline 里写过 haiku 顶端给分偏松（把
  2024b 打到 99.3）。一个全是优秀论文的集合抓不到"高估"——把优秀论文打成优秀，看
  着一切正常。绝对分校准在这种数据上做不了。

  但它仍有两个扎实的用法

  A. 天花板 / 假阴性检查（直接可用）。
  优秀论文就该高分。如果评委把一篇国赛优秀论文打到
  <70，那是一个明确可抓的错（假阴性）。再加一层：改 prompt /
  换模型后，优秀论文的分不应该掉——这就是一个回归基线。这能测"会不会冤枉好论文"，
  测不了"会不会捧高烂论文"。

  B. 合成对照（扰动法）—— 这才是解你约束的钥匙。
  把一篇优秀论文按单一维度"做坏"，生成一个已知应当更低的变体，于是你白得一对
  原版 > 劣化版 的 ground-truth 
  排序，完全不需要真实的中/劣论文。而且劣化方式可以精准对准我们关心的维度：

  ┌─────────────────────────────────┬─────────────────────┬─────────────────┐
  │              扰动               │  应触发的扣分维度   │    顺带验证     │
  ├─────────────────────────────────┼─────────────────────┼─────────────────┤
  │ 删符号表 + 注入 2–3             │ 模型合理性/严谨性   │ 符号检查器应报  │

  ├─────────────────────────────────┼─────────────────────┼─────────────────┤
  │ 删掉灵敏度/鲁棒性章节           │ 灵敏度分析          │ —               │
  ├─────────────────────────────────┼─────────────────────┼─────────────────┤
  │ 把可追溯数字替换成改过的/无源数 │ 求解正确性/结果说服 │ verify_numbers  │
  │ 字                              │ 力                  │ 的 UNMATCHED    │
  │                                 │                     │ 应升            │
  ├─────────────────────────────────┼─────────────────────┼─────────────────┤
  │ 抽掉关键推导步骤（制造逻辑跳跃  │ 模型合理性/写作     │ （PDF 的 AST    │
  │ ）                              │                     │ 思路）          │
  ├─────────────────────────────────┼─────────────────────┼─────────────────┤
  │ 打乱一张结果表                  │ 结果说服力          │ —               │
  └─────────────────────────────────┴─────────────────────┴─────────────────┘

  这正是 NLP 里成熟的反事实/扰动测试（CheckList、对比评估）思路，并且劣化的恰好
  是真实中/劣论文最常见的缺陷——所以合成出来的"低端"是有代表性的，不是玩具。

  为什么扰动法对我们三条线都完美

  - 能量化、能提交：指标 =
  "定向扣分正确率"——劣化版在目标维上显著低于原版的比例（N 篇 × M 种扰动 = N·M
  个有向 pair）。这是一个真数字，能进 git（只交指标，不交论文）。
  - 验证多模型面板：比 haiku / Gemini-Flash / DeepSeek-Flash
  各自的"定向扣分正确率 + 方差"——谁更能稳定地把劣化版判低、且扣在对的维度，一目
  了然。这把上一轮"加模型有没有用"从口水变成可测。
  - 验证符号检查器：注入未定义符号的变体里，verify_symbols
  抓没抓到、评委严谨性维有没有跟着扣——直接闭环。

  现实 / 合规

  - 优秀论文有版权 → 本地放、gitignore，只把派生指标/分数提交（这正是 plan

  残留缺口（仍要承认）

  扰动法测的是向下检测能力（能不能识别缺陷），仍**不能直接测"会不会把平庸论文捧
