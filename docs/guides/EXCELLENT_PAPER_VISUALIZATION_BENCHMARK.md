# CUMCM 优秀论文可视化基准

本基准来自官方优秀论文展示的跨年 A 题样本：2024A A163/A242/A016/A053、2025A A196、2023A A0165。2025 赛题讲评材料只能作为扣分点参考，不作为优秀论文样本。

## 核心判断

优秀论文的图不是越多越好，而是每张图承担明确叙事任务。图表主要服务三件事：

1. 降低模型理解门槛：解释几何关系、物理机理、变量定义、区域划分。
2. 支撑最终结果可信度：展示关键状态、最紧约束、搜索过程、收敛性或验证对比。
3. 压缩大量数值结果：表格报告精确值，图展示趋势、空间分布和机制。

## 图表角色

每张进正文的图或表必须标注一个主要叙事角色：

- `explain_model`：解释模型对象、几何关系、物理机理或子问题依赖。
- `report_result`：报告某个子问题的最终结果或关键状态。
- `validate_result`：支撑最终口径可信度，如收敛、敏感性、算法对比、最紧约束。
- `show_limitation`：展示局限、未采信分支或失败模式。此类图必须后置到灵敏度、模型评价或附录。

正文主线优先使用前三类。`show_limitation` 不得抢占每问主图位置。

## 选图规则

### 1. 复杂公式前先给解释图

几何、路径、碰撞、遮挡、光学、动力学等复杂判据，在公式推导前应优先配一张概念图。图中只标关键对象、方向、区域、边界和判定关系，不展示无关中间变量。

### 2. 每个子问题要有一个视觉锚点

每个子问题至少有一个主图或主表作为最终交付视觉锚点。可以是：

- 精确结果表。
- 关键状态图。
- 最优路径/方案图。
- 结果趋势图。
- 算法对比表。

不要只在正文中散落数字。

### 3. 几何路径题使用三件套

路径规划、碰撞检测、调头、车辆/链式运动类问题优先使用：

- 全局路径图。
- 关键临界态或最紧约束图。
- 局部放大或多时刻快照。

全局路径图用于理解方案，临界态图用于证明约束，局部放大图用于解释最紧关系。

### 4. 优化搜索题必须展示可信度

若最终答案来自搜索、启发式算法、一维优化、网格扫描或数值迭代，应至少提供下列之一：

- 目标函数曲线，并标出最优点。
- 可行边界或约束裕度曲线。
- 收敛曲线。
- 独立算法对比表。

这类图的 caption 必须说明“为什么最终结果可信”。

### 5. 空间分布题使用统一多宫格

若模型输出是位置、节点、网格、城市、镜场等空间单元上的指标，优先用统一坐标、统一或可比色标的多宫格散点图/热力图。每个子图表示一个指标，读者应能横向比较空间模式。

### 6. 图和表分工明确

表格报告精确数值、参数设计、策略清单、附件结果；图展示趋势、机制、空间分布、临界状态和可信度。不要用图替代表格中的精确值，也不要用大表格替代趋势图。

## 不推荐

- 不强制使用现实照片。照片只适合题目背景，不是核心建模证据。
- 不画 runner、workflow、缓存、文件路径等内部工程流程图。
- 不把粗网格下界、退化解、失败分支作为主图。它们只能作为验证、局限或附录图。
- 不为了凑数量画图。每张图必须服务一个明确结论。

## `visualization_log.md` 要求

Step 8 的 `visualization_log.md` 图表清单必须包含：

- 图文件。
- 论文章节。
- 子问题。
- 叙事角色：`explain_model` / `report_result` / `validate_result` / `show_limitation`。
- caption 起草。
- 依据来源：`solve_log.md`、`sensitivity_report.md`、`results/*`、`model.md` 或 `problem/*`。

caption 不只描述“图中有什么”，还要写出“这张图支持什么结论”。
