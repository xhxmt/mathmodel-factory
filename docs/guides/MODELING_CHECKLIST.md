# 建模口径纠错清单（方法论版 · 无本题答案）

> 用途：Step 4 模型构建 / `modeling_scope_gate.md` 的口径自检。本文件**只讲方法论**，
> **不含任何具体题目的答案数值或范文基准**——这是刻意的：运行时 agent 参考本题范文答案
> 等于开卷考试，会污染 benchmark（见 `docs/RERUN0706_REPAIR_PLAN.md` §L5 / §E）。
> 需要用范文答案做校准是**赛后离线评估**的事，在 `evaluation/answer_keys/` 下进行，
> 不在运行时。以下每条都能从**题面本身**推导，无需外部答案。

## 一、遮蔽/覆盖/服务类问题的判据口径

- **整体 vs 任一点**：目标是"真目标（圆柱/区域/多点集合）**被完全**遮蔽/覆盖"时，
  判据必须是"**所有**采样点均满足"（合取），不是"**任一**采样点满足"（析取）。
  析取会系统性高估。若确实允许部分遮蔽，必须有题面或文献支持并做阈值实验
  （50%/75%/100% 采样点）。
- **采样充分性**：离散采样近似连续目标时，必须做**采样密度收敛实验**（如每层
  16→50→100+ 点），证明目标量随加密单调收敛且采纳网格的相对误差可接受。
  能证明"只需检查边界（如上下底面圆周）即可判定整体"时，先证明再采样，省一个量级。
- **线段 vs 无限直线**：视线遮蔽判据用**线段**，不用无限延长线——后者会在端点外
  虚增遮蔽（可虚增数倍）。
- **精确的线段–球相交，不是"垂足 + 垂距"近似**：正确判据是解一元二次
  `‖m + s·(p−m) − c‖ = r` 的根 s₁ ≤ s₂：判别式 ≥ 0 且（某根落在 [0,1]，**或**
  整段在球内 s₁<0 ∧ s₂>1）。**反面案例**（看似等价、实则漏判）：只查"垂足在
  线段内（`0<proj<‖L‖`）且垂距 ≤ r"——垂足滑出线段端点、但球体仍切割线段
  末端的构型会被误判为未遮蔽，系统性**低估**遮蔽区间的边界（实测量级 ~2%，
  且总在区间出入口处）。**判据定稿前必须做双实现交叉校验**：另写一份独立的
  精确判据实现，对同一固定策略（如题面给定参数的第 1 问）复算，两实现差值
  超过 1e-3 时以精确判据为准并回修模型公式。
- **精度口径（canonical 采信值必须精化）**：粗时间网格（Δt≈0.05 s 级）只准
  用于**搜索定位**；canonical 采信值必须对每个遮蔽区间做**端点二分精化**
  （到 1e-3 s 级）后按精化区间并集统计，粗网格计数只能写进诊断字段。红旗
  信号：采信值恰为 Δt 的整数倍 = 粗网格计数被当成 canonical、精化值被降级成
  "诊断"——方向反了（`verify_invariants.py` 的 `quantized_off` 不变量专抓这个）。

## 二、多对象 / 多资源目标函数

- **交集 vs 求和 vs 并集**——先分清题意问的是哪一个，三者物理含义完全不同：
  - "同时保护多个对象 T 秒" → 多对象遮蔽区间**交集**长度（∩）。
  - "各对象分别被保护的总时长" → 各对象**并集之和**（Σ over objects of union）。
  - **单一对象、多枚资源** → 各资源遮蔽区间的**并集**（∪），不能把各资源时长直接
    相加（重叠时段会重复计分）。
  - 选定口径必须在 `modeling_scope_gate.md` 写明依据，并把不采用的口径作为**诊断量**
    同时报告（如把"同时遮蔽交集"作为更强约束的诊断），供决策者对照。
- **区间并集消重**：任何"多枚资源总时长"都必须用区间并集测度实现，写一条不变量
  `总量 ≤ 各分项之和`（`type: le`）自查是否重复计分。

## 三、高维 / 协同 / 分配问题的求解结构

- **先做结构分析再上元启发式**：≥8 维或含离散分配的子问题，直接扔给 DE/PSO/GA
  往往陷局部峰。优先论证三类降维（至少其一），否则 critic 打回：
  1. **判据简化证明**：能否证明只需检查边界/极值点（降评估成本）。
  2. **几何降维**：能否用几何关系（如预测拦截点/关键相遇时刻）把决策维度压下来，
     再局部精修。
  3. **分层分解**：能否"先分配后优化"（先定哪资源打哪目标，再各自连续优化）。
- **增益必须可证**：多资源/协同方案的目标必须**严格优于**少资源基线（`gt_strict`
  带正 margin，不是 `ge`）；任何"加了资源但零增益/某资源零贡献"都是搜索失败信号
  （用 `nonzero` 不变量抓），要加预算重搜或书面论证，不能当成"结论"。
- **逐资源非零贡献（批量）**：≥3 枚资源的方案，canonical 必须落
  `per_smoke[].contribution`（每枚资源**单独**评估的贡献），并声明
  `nonzero_each` 不变量逐枚检查。总量 T_i>0 挡不住"9 枚里 4 枚零贡献"——
  按导弹/对象粒度自检的漏网正是按资源粒度能抓到的。
- **预算与停机**：全局搜索给足预算（迭代/评估不低于门禁下限，且评估数下限随
  维度增长——12 维和 4 维不能用同一阈值），并跑预算阶梯证明边际收益已
  flatten。**警惕假收敛**：弱搜索器的平台期（连续两档预算目标不动）≠ 全局
  最优——若某已部署资源零贡献、或与结构 baseline（几何引导初值的局部搜索）
  差 >5%，平台期只说明"这个搜索器搜不动了"。
- **松弛上界**：给一个可计算的宽松上界（去耦合独立最优和 / 资源寿命上界 / LP 松弛），
  用 gap 诚实标注解质量——这是**自证**，不依赖任何外部答案。

## 四、与 demo / 完整规模的连续性

- Step 4 要把 demo 规模的 sanity 数值列出，给完整规模的**预期偏差区间**（从模型结构
  推，不是从范文答案推），Step 5 完整求解后对照，超出即触发 audit。
- 完整规模的采样密度、搜索预算必须 ≥ demo；`model.md` 声明的规模必须与 Step 5 实际
  运行参数一致（不得声明 N=150 却跑 N=51）。

## 使用方式

在 `modeling_scope_gate.md` 用本清单逐条给 PASS/FAIL/N/A + 依据。不适用的题型写明理由。
**注意**：本清单所有判断都从题面结构导出；若你发现自己在引用某个"应该是多少秒"的
外部基准数值，停下——那是答案污染，不是建模。
