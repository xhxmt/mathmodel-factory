## 共性判断

结论：2025A A196 与 2023A A0165 支持此前 2024A 对标报告中的主要写作判断。这些特征不是 2024A 板凳龙题目的个案，而是跨年度 A 题优秀论文的稳定写法。

### 1. 摘要按子问题交付，而不是先展示系统框架

2025A A196 的摘要从第 1 问到第 5 问逐段写作，每段都包含“模型/算法 -> 关键结果 -> 验证或附件”。2024A A242 也是同一结构，明确写出 P2 停止时间、P3 最小螺距、P5 最大速度等关键数字。2023A A0165 是三问结构，摘要用“对于给定镜场参数情形”“对于统一定日镜尺寸及安装高度情形”“对于各定日镜尺寸及安装高度可变情形”逐问组织，实质仍是子问题摘要。

接入规则：CUMCM 摘要不应固定为“问题理解/方法/结果/亮点”四段散文。如果题目分问明确，应采用“开头总述 + 每问一段”的优秀论文结构；四段法的四个功能可以分散进逐问段落中。

### 2. 问题分析使用“难点 -> 建模对象 -> 方法 -> 输出”的短索引

2025A A196 在“问题分析”中按问题一至问题五分别说明：该问的核心困难、决策变量或几何关系、优化/搜索方法、最终要求输出。2024A A242/A016 也采用这种结构。2023A A0165 对三问分别解释指标计算、统一尺寸优化、可变尺寸优化，段落短，目的明确。

接入规则：Step 9 起稿时，问题分析章节必须按子问题写“阅卷索引句”，不要把 AABB、SAT、NLP、遗传算法、工作流文件名等工具链堆在章节开头。

### 3. 主文先给最终采信口径，诊断和风险后置

优秀论文通常先报告最终答案，再解释算法和验证。2025A A196 在 P1/P2/P4 中先给有效遮蔽时长、投放策略、最大遮蔽时间，再用敏感性分析或替代算法验证。2024A A242 在 P3 摘要直接报 `0.450337m`，不会让粗网格下界抢占主叙事。

接入规则：模型求解章节每问都要先给最终采信结果。粗网格、下界、乐观上界、失败分支、退化分支、fallback 等只放在验证、灵敏度或局限性中。

### 4. 验证用于支撑可信度，不用于放大不确定感

2025A A196 多次使用独立算法或敏感性分析支撑结论，例如 P1 的圆周离散点数敏感性、P2 的粒子群复核、P4 的差分进化复核。2023A A0165 设置“模型检验”和“模型评估”，用太阳视运动规律和对称性解释模型合理性。2024A 对标报告中提到的 P3/P5 加密验证也应写成“为什么最终值可靠”，而不是写成“模型脆弱”。

接入规则：灵敏度章节标题和措辞应以“加密验证、收敛性、独立复核、稳定性”为主，必要风险放入局限性；摘要和结论只写最终采信口径。

### 5. 附件和结果文件可出现，但内部工程痕迹不能进入正文

优秀论文会提 `result1.xlsx`、`result2.xlsx` 等题目要求附件，但不会出现内部模型别名、JSON 路径、runner 状态、fallback 模板、缓存、workflow、RELAXED 等工程语言。2024A 当前稿的问题正是把这些系统审计痕迹放进了主叙事。

接入规则：Step 12/15 必须把工程痕迹改写为论文语言。允许保留题目要求的 `result*.xlsx`；不允许在正文主线中出现 `m1/m2`、`results/*.json`、`RELAXED`、`fallback`、`workflow`、`solver_submit.sh`、`cache` 等内部过程词，除非在附录代码清单中必要出现。

### 6. 结论按题号列最终答案，不重复调试路径

2023A A0165 和 2025A A196 的结论/评估不再重述所有搜索失败过程，而是归纳模型优缺点和最终方案。2024A 优秀论文的结论和模型评价也保持“最终答案 + 可信依据 + 局限”的顺序。

接入规则：结论章节必须按子问题列最终推荐值和一条可信依据。不要把粗网格下界、退化路径、未采信分支和调试文件名放进结论。

## 对 2024A 改稿规则的加强

上述共性强化了 `docs/reference/2024A_writing_comparison.md` 的核心建议：

1. 摘要改成五问逐段结构。
2. P3 主答案采用加密验证后的 `0.450m` 口径，粗网格 `0.429883m` 后置为下界。
3. P4 先讲几何构造与最终路径，再讲全队无碰撞核验和局限。
4. P5 必须报告峰值区间、峰值把手、速度倍率和搜索步长。
5. 灵敏度从“暴露风险”改成“支撑最终采信口径”。
6. 删除正文中的内部工程痕迹。
