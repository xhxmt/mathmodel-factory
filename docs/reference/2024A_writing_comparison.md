## 逐项差距与改法

### 1. 摘要：从“四段技术摘要”改成“五问阅卷摘要”

你的摘要现在是“问题理解、方法、结果、亮点”四段法，内容完整，但前两段技术密度太高，评委要读到第三段才集中看到五问答案。A242/A016 都是“针对问题一/二/三/四/五”逐段命中。


### 2. 问题分析：减少“技术堆叠”，每问先给阅卷索引

你的问题分析已经完整，但第一眼看到的是 AABB、SAT、Brent、NLP 等工具链。优秀论文更常见的写法是每问开头先回答四件事：难点是什么、建模对象是什么、采用什么方法、输出什么结果。

建议把问题分析改成每问一段“索引句 + 方法句”。示例：

```tex
问题二的核心在于确定舞龙队沿螺线继续盘入时的首次实体碰撞时刻。由于碰撞可能发生在两个整数秒之间，本文不直接以逐秒结果判定，而是把任意非相邻板凳矩形间的最小距离定义为连续裕度函数 $g(t)$，先用 AABB 缩小候选板凳对，再用 SAT 计算精确距离，并用 Brent 法求解 $g(t)=0$ 的最早时刻。
```

```tex
问题三的核心在于在“能够进入直径 $9\,\mathrm{m}$ 调头空间”和“进入前全队不碰撞”之间寻找最小螺距。本文以螺距 $p$ 为决策变量，先由龙头到达 $r=4.5\,\mathrm{m}$ 确定检查区间，再在该区间内复用问题二的实体碰撞检测模型判断可行性，最后用一维搜索得到推荐螺距。
```

```tex
问题五的核心在于局部曲率变化会放大后续把手速度。本文先固定龙头速度为 $1\,\mathrm{m/s}$，计算全路径、全把手的速度放大倍率 $A=\max v_i/v_0$，再利用速度线性比例关系反解 $v_0^\star=2/A$。
```

### 3. 问题三：主文只报 `0.450m`，不要让 `0.429883m` 抢叙事

你的 P4 写作最大问题是“退化分支、NLP 塌缩、Rmin 降格、无碰撞裕度”等解释过早出现。优秀论文通常先把 P4 写成几何构造题：相切条件、半径比、圆弧长度、能否更短，然后才给可行性验证。

建议 P4 主文开头改为：

```tex
R_1=3.005418\,\mathrm{m},\quad R_2=1.502709\,\mathrm{m},\quad

两段圆弧均与相邻曲线保持切向连续，整条调头曲线位于半径 $4.5\,\mathrm{m}$ 的调头空间内。
```

关于“能否更短”，建议主文只保留：

```tex
在保持“与盘入、盘出螺线相切”和“半径比 $2{:}1$”两个题给条件不变时，圆弧半径和切点位置由几何方程唯一确定，因而曲线长度不能继续通过调节圆弧缩短。若放松题给相切或半径比约束，可得到数学上更短但无法承载整条板凳链通过的退化路径，故不作为本题可执行方案。
```

全队核验建议紧跟其后，写成加分项：

### 5. 问题五：必须解释 `1.258747m/s` 与优秀论文常见 `1.246266m/s` 的差异

A163/A242 都给出约 `1.246266m/s`，你的 `1.258747m/s` 很接近但不同。这个差异可以接受，但必须在正文给出“峰值区间、峰值把手、倍率、搜索步长”的说明，否则评委会怀疑是采样口径不同。

建议把 P5 主文改成：

```tex
由于各把手速度与龙头速度成线性比例，先令龙头速度 $v_0=1\,\mathrm{m/s}$，计算问题四路径上所有把手的速度放大倍率
\[
A=\max_{i,t}\frac{v_i(t)}{v_0}.
\]
粗步长扫描显示最大速度出现在调头后 $14$--$15\,\mathrm{s}$ 附近。进一步将时间步长加密到 $0.5\,\mathrm{s}$ 并复核到 $0.25\,\mathrm{s}$，最大倍率稳定为 $A=1.588882$，峰值出现在 $t=14.5\,\mathrm{s}$、第 3 号把手附近。于是龙头最大安全速度为
\[
v_0^\star=\frac{2}{A}=1.258747\,\mathrm{m/s}.
\]
```

然后补一句解释差异：

```tex
已有优秀论文中常见的 $1.246266\,\mathrm{m/s}$ 结果对应其调头路径与峰值搜索口径。本文采用非退化 S 形曲线并对全把手速度倍率做加密扫描，得到的峰值倍率略小于该口径，因此反解速度略高。由于 $0.5\,\mathrm{s}$ 与 $0.25\,\mathrm{s}$ 结果一致，本文采用 $1.258747\,\mathrm{m/s}$ 作为本模型下的推荐安全速度。
```

如果后续重新核算发现路径口径应与 A242/A163 一致，则应优先修正数值，而不是强行解释差异。

## 2025 讲评可借鉴的扣分点

2025A 讲评材料不是优秀论文样本，但其写法提醒了一个共通扣分点：很多队伍只用离散步长做判定，容易把“采样点满足”误写成“连续过程满足”。这正好对应你的 P3 和 P5。

因此你的论文里要避免三种写法：

1. 不要说“逐秒检查无碰撞，所以全程无碰撞”。
2. 不要把粗步长搜索得到的 `0.429883m` 或 `1.443369m/s` 当最终答案。
3. 不要只给程序结果，要说明加密/收敛/连续定位如何支撑最终口径。

推荐保留的表述是：

```tex
由于碰撞与速度峰值都可能发生在离散采样点之间，本文对关键结论采用加密抽样或连续事件定位进行复核，并以复核后的保守值作为最终提交结果。
```

## 跨年优秀论文共性验证

补充对 2025A 优秀论文 A196 与 2023A 优秀论文 A0165 的 OCR 对比后，上述写作建议仍成立，并可视为跨年度 A 题优秀论文共性，而不是 2024A 板凳龙题目的偶然特征。详见 `docs/reference/2023_2025A_writing_commonality.md`。

主要共性包括：

1. 摘要按子问题逐段交付，通常是“模型/算法 -> 关键结果 -> 验证或附件”，而不是先写大段系统框架。
2. 问题分析按题号写短索引，先说明难点、建模对象、方法和输出。
3. 主文先报最终采信口径，粗网格、失败分支、退化解和风险解释后置。
4. 验证和灵敏度分析用于支撑最终结论的可信度，不在摘要和结论中放大不确定感。
5. 正文保留题目要求的 `result*.xlsx` 附件说明，但不出现内部工程路径、runner 状态、fallback、缓存或内部模型别名。

这些规则已接入系统流程：`docs/guides/EXCELLENT_PAPER_WRITING_BENCHMARK.md` 作为共享基准，并由 Step 9、Step 11、Step 12、Step 13、Step 14、Step 15 的 prompts 引用。
