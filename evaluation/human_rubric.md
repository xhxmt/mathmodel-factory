## 各维度评判要点

**1. 模型合理性 (20)** —— 假设是否明确且站得住？符号是否在符号表中齐全？子问题之间衔接是否自洽？建模与题目要求是否对得上？

**2. 求解正确性 (20)** —— 数值是否可在 logs/results 中追溯（见 `verify_numbers.py` 的 UNMATCHED 计数）？求解器 / 算法选择是否恰当？有无明显计算错误或不收敛？

**3. 创新性 (20)** —— 相对最朴素做法（枚举 / 单一评价模型）是否有清晰的区分度？亮点是团队设计还是文献既有？是否被论文讲透？

**4. 写作清晰度 (15)** —— 章节结构是否完整（摘要 / 重述 / 分析 / 假设 / 符号 / 建立 / 求解 / 灵敏度 / 评价 / 参考 / 附录）？图表是否自洽？语言是否流畅、无机器味？

**5. 结果说服力 (15)** —— 关键结论是否有表格 / 图形支撑？有无对照基准？数字与口径是否诚实（不夸大）？

**6. 灵敏度分析 (10)** —— 扰动设计是否覆盖关键参数？结论在扰动下是否稳健？翻转点是否被诚实披露？
