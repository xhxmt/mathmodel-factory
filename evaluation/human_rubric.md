## 各维度评判要点

**1. 模型呈现 (20)** —— 在硬有效性已经通过的条件下，假设、变量、约束和子问题衔接是否被清楚、完整地呈现？

**2. 求解叙事 (20)** —— 求解路径、算法选择、参数口径和结果生成过程是否讲清楚，使读者能够理解并复核？这里评价叙事质量，不重复给数学正确性打分。

**3. 创新性 (20)** —— 相对最朴素做法（枚举 / 单一评价模型）是否有清晰的区分度？亮点是团队设计还是文献既有？是否被论文讲透？

**4. 写作清晰度 (15)** —— 章节结构是否完整（摘要 / 重述 / 分析 / 假设 / 符号 / 建立 / 求解 / 灵敏度 / 评价 / 参考 / 附录）？语言是否流畅、无机器味？文本对齐模式只根据图题、表题、标签和正文引用判断叙事是否自洽，不评价实际渲染美观度。

**5. 结果说服力 (15)** —— 关键结论是否在正文、图题 / 表题和引用关系中得到表格 / 图形支撑？有无对照基准？数字与口径是否诚实（不夸大）？文本对齐模式不猜测图像像素中不可见的信息。

**6. 敏感性与局限 (10)** —— 扰动设计是否覆盖关键参数？稳定性、适用边界、失败条件和限制是否被诚实解释？
