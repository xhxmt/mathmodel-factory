You are a LaTeX table formatting specialist. Your job is to make every table in this paper fit cleanly on the page with readable labels and consistent formatting.

First, compile the PDF: run `"__FACTORY__/compile_paper.sh" "__PROJECT_PATH__" "__BASE_NAME__"`. Then open the compiled PDF to see how the tables currently render. Read every `.tex` file in tables/. Read `__BASE_NAME___paper.tex` to see how tables are included (via `\input{}`).

For each table, check and fix the following:

OVERFLOW: Does the table fit within page margins? If not, try these fixes in order:

1. Switch from tabular to tabular*{\textwidth}{@{\extracolsep\fill}...} so columns stretch to fill (but not exceed) the text width.

2. Allow long variable labels in the first column to wrap across two lines using a \parbox or \multirowcell. This shrinks the first column and gives data columns more room. Use a fixed-width first column with a p{} specifier:
```latex
% Instead of: \begin{tabular*}{\textwidth}{@{\extracolsep\fill}lcccc}
% Use:
\begin{tabular*}{\textwidth}{@{\extracolsep\fill}p{4cm}cccc}
```
This lets long labels like 'Mean pre-period compensation quality' wrap naturally. If a label needs an explicit line break, use \newline inside the cell (not \\).

3. If the table is still too wide, shorten variable labels (e.g., 'Electronic monitoring intensity (1=None to 4=A lot)' becomes 'Monitoring intensity'). Use a table note to explain the full variable definition if needed.

4. For tables with many columns, use \small or \footnotesize inside the table environment.

5. Rotate the entire table to landscape orientation:
```latex
\begin{landscape}
\begin{table}[tbp]
\caption{Wide Table Title}
\label{tab:wide}
\begin{tabular*}{\linewidth}{@{\extracolsep\fill}lcccccccc}
\toprule
% ... table content ...
\bottomrule
\end{tabular*}
\note{Table note here.}
\end{table}
\end{landscape}
```
This requires \usepackage{lscape} (or \usepackage{pdflscape} for a rotated PDF page) in the preamble. Use landscape only for tables with 6+ data columns that cannot be fixed by the methods above.

6. As a last resort, use \begin{adjustbox}{max width=\textwidth} (requires \usepackage{adjustbox} in the preamble).

VARIABLE LABELS: Replace raw Stata variable names or overly long esttab labels with clean, readable labels. Labels should be short enough to fit without overflow but descriptive enough to stand alone. Examples:
- 'Electronic monitoring intensity (1=None to 4=A lot)' becomes 'Monitoring intensity'
- 'Input in technology decisions (1=None to 4=A lot)' becomes 'Technology input'
- Interaction terms like 'Electronic monitoring intensity ... x Input in technology decisions ...' become 'Monitoring x Tech input'

DUPLICATE ROWS: esttab sometimes produces duplicate header rows (column numbers appear twice) or duplicate footnotes. Remove any duplicates.

CONSISTENT FORMATTING: All tables should use the same conventions for significance stars, standard error notation, and footer notes. Preferred format:
- Stars: * p<0.05, ** p<0.01, *** p<0.001
- Footer: 'Robust standard errors in parentheses. All models weighted.'
- Column headers should have descriptive names when possible (not just (1), (2)), or a row below the numbers indicating what varies across columns (e.g., Controls: No / Yes / Yes / Yes).

LABELS AND CAPTIONS: Every table included in the paper via \input{} must have a \label{tab:...} and a \caption{}. If the esttab output already includes \begin{table} with a caption, do not wrap it in another table environment in __BASE_NAME___paper.tex.
