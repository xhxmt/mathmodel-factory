EVENT STUDY DESIGN AUDIT (if the analysis includes event studies or staggered DiD):
The event study pre-period is critical for credibility. Audit the following and fix any issues directly:

1. PRE-PERIOD LENGTH: The event window should include at least 3 pre-treatment periods where possible, but weight this against sample size reductions when using a balanced sample. The event window does NOT need to match the treatment definition window — even if identifying treatment requires only 2 lags, the event study can and should extend further back. In a stacked design, later cohorts will naturally have more pre-period data than early cohorts; use all of it. If some cohorts have short pre-periods due to data availability, that is fine — include them with whatever pre-period they have, and let the regression estimate what it can.

2. REFERENCE PERIOD: The omitted reference period should be t-1 (the period immediately before treatment). Verify this is what the code uses. If the code omits a different period, or if the reference period is ambiguous, fix it.
