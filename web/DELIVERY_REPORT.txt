🚀 使用说明 (Usage Instructions)
═══════════════════════════════════════════════════════════════════

快速启动
────────────────────────────────────────────────────────────────
  1. 进入目录：
     $ cd /home/tfisher/paper_factory/web

  2. 一键启动：
     $ ./start_dashboard.sh

  3. 访问浏览器：
     http://localhost:5173

  4. 停止服务：
     按 Ctrl+C

创建演示项目（测试用）
────────────────────────────────────────────────────────────────
  $ cd /home/tfisher/paper_factory/web
  $ ./create_demo_projects.py

  这会创建 5 个演示项目：
  - demo_running_step3       (运行中，步骤 3)
  - demo_consultation_step4  (等待咨询，步骤 4)
  - demo_paused_step7        (已暂停，步骤 7)
  - demo_completed           (已完成，步骤 16)
  - demo_early_stage         (运行中，步骤 1)

清理演示项目
────────────────────────────────────────────────────────────────
  $ rm -rf ongoing/demo_* complete/demo_*

💡 使用场景示例 (Use Cases)
═══════════════════════════════════════════════════════════════════

场景 1：批量实验监控
────────────────────────────────────────────────────────────────
  # 启动多个 ablation 实验
  $ ./launch_agents.sh new exp1 /path/to/problem.pdf
  $ ./launch_agents.sh new exp2 /path/to/problem.pdf
  $ ./launch_agents.sh new exp3 /path/to/problem.pdf

  # 打开 Dashboard 监控所有实验
  $ cd web && ./start_dashboard.sh

  在浏览器中一目了然地看到：
  - 每个实验的当前步骤
  - 进度条和完成百分比
  - 是否有失败或卡住的任务

场景 2：人工咨询介入
────────────────────────────────────────────────────────────────
  当项目到达 consultation gate：

  1. Dashboard 自动高亮（黄色 "等待咨询"）
  2. 点击项目卡片 → "查看详情"
  3. 切换到 "人工咨询" 标签
  4. 复制问题到 GPT Pro / Gemini Deep Think
  5. 粘贴回答并提交
  6. 项目自动恢复运行

场景 3：调试和日志查看
────────────────────────────────────────────────────────────────
  当某个步骤失败时：

  1. 在 Dashboard 看到项目停止在某一步
  2. 点击 "查看详情" → "日志" 标签
  3. 查看最近 100 行日志
  4. 定位 solver 错误或数据问题
  5. 修复后手动恢复项目
