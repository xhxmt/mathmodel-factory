# Web Dashboard 界面说明

## 主界面（项目列表）

Dashboard 主界面展示所有项目的概览信息：

### 顶部统计栏
- **运行中**：当前正在执行的项目数量（蓝色）
- **等待咨询**：需要人工介入的项目数量（黄色高亮）
- **已完成**：已完成的项目数量（绿色）

### 项目卡片

每个项目卡片显示：

1. **状态指示器**：
   - 🔵 蓝色圆点跳动：运行中
   - 🟡 黄色圆点跳动：等待咨询
   - 🟢 绿色圆点：已完成
   - ⚪ 灰色圆点：已暂停

2. **控制按钮**：
   - ⏸ 暂停：暂停正在运行的项目
   - ▶️ 恢复：恢复已暂停的项目
   - ⏹ 终止：强制终止项目（需确认）

3. **进度条**：
   - 显示当前步骤（例如：步骤 5/16）
   - 可视化进度百分比

4. **咨询提醒**：
   - 当项目需要人工介入时，显示黄色警告框
   - 标明等待的 gate（例如：preflight、step4）

5. **元数据**：
   - 最后更新时间（相对时间或绝对日期）
   - 进程 PID（如果正在运行）

## 项目详情弹窗

点击 "查看详情" 按钮打开详情弹窗，包含三个标签页：

### 1. 概览 (Overview)

显示项目的核心信息：

- **状态网格**：
  - 当前状态（运行中/暂停/完成/等待咨询）
  - 当前步骤（X/16）
  - 进度百分比
  - 进程 PID

- **Checkpoint 内容**：
  - 显示 `checkpoint.md` 的完整内容
  - 包含步骤描述、时间戳等

### 2. 日志 (Logs)

查看项目的最新执行日志：

- 默认显示最近 100 行日志
- 点击 "刷新" 按钮获取最新日志
- 日志以终端风格显示（等宽字体、暗色背景）
- 自动滚动到底部

**适用场景**：
- 调试 solver 错误
- 查看步骤失败原因
- 监控长时间运行的任务

### 3. 人工咨询 (Consultation)

处理人工介入请求：

#### 当无待处理咨询时
显示：✅ 当前无待处理的咨询请求

#### 当有待处理咨询时

**咨询卡片头部**：
- 标题：例如 "启动前 seed：题目解读与候选方法"
- Badge：gate 名称（preflight、step4、dynamic）

**元数据**：
- 步骤编号
- 创建时间

**需要决定的事项**：
- 显示咨询请求的具体内容
- 通常包含需要决策的问题或选项

**提交回答表单**：
1. 提示文字：
   > 请将 GPT Pro / Gemini Deep Think 的结论粘贴到下方：

2. 多行文本框：
   - 预填充示例格式
   - 支持 Markdown 格式
   - 建议包含：
     - 分析标题（例如：## 建模方案分析）
     - 具体建议（主模型、辅助方法、参数选择）
     - 理由说明

3. 提交按钮：
   - 点击 "提交并恢复运行"
   - 自动写入 `human_review.md` 并标记 `STATUS: READY`
   - 关闭弹窗并刷新项目列表

## WebSocket 实时更新

Dashboard 使用 WebSocket 保持与后端的持久连接：

- **自动推送**：项目状态变化时立即更新 UI，无需手动刷新
- **连接状态**：顶部显示连接状态（断开时显示黄色警告）
- **自动重连**：连接断开后每 3 秒自动尝试重连

## 颜色语义

| 颜色 | 含义 | 使用场景 |
|------|------|----------|
| 🔵 蓝色 | 正在运行 | 状态点、进度条、统计数字 |
| 🟡 黄色 | 需要注意 | 咨询请求、等待人工介入 |
| 🟢 绿色 | 成功/完成 | 已完成项目、成功消息 |
| ⚪ 灰色 | 暂停/不活跃 | 已暂停项目、元数据文字 |
| 🔴 红色 | 错误/终止 | 失败状态、终止按钮 |

## 快捷键和交互

- **ESC**：关闭详情弹窗
- **点击背景**：关闭详情弹窗
- **卡片悬停**：轻微上浮效果，增强边框
- **按钮悬停**：颜色加深，显示 tooltip

## 响应式设计

Dashboard 支持不同屏幕尺寸：

- **桌面端（>768px）**：
  - 项目卡片：多列网格布局（自适应列数）
  - 详情弹窗：固定宽度（最大 900px）

- **移动端（≤768px）**：
  - 项目卡片：单列布局
  - 顶部统计：垂直堆叠
  - 详情弹窗：全宽显示

## 浏览器兼容性

推荐使用现代浏览器：
- Chrome/Edge 90+
- Firefox 88+
- Safari 14+

需要支持：
- WebSocket API
- CSS Grid
- ES6 模块

## 性能说明

- **更新频率**：WebSocket 每 2-3 秒推送一次状态更新
- **日志加载**：默认 100 行，可按需刷新
- **内存占用**：单个会话通常 < 50MB
- **网络流量**：每小时约 1-2MB（主要是 WebSocket 心跳）
