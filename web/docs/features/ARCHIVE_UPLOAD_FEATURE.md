# 压缩包上传功能说明

## 概述

前端和后端已升级，支持上传包含题目和数据文件的压缩包，系统会自动解压并识别题目文件。

## 支持的格式

### 压缩包格式
- `.zip` - ZIP 压缩包
- `.tar.gz` / `.tgz` - Gzip 压缩的 tar 归档
- `.tar.bz2` - Bzip2 压缩的 tar 归档
- `.tar.xz` - XZ 压缩的 tar 归档
- `.tar` - 未压缩的 tar 归档

### 题目文件格式（压缩包内）
- `.pdf` - PDF 格式题目
- `.md` - Markdown 格式题目

## 工作流程

### 1. 用户上传
用户可以通过前端界面上传：
- **单个题目文件**：PDF 或 Markdown
- **压缩包**：包含题目文件和相关数据的压缩包

### 2. 后端处理

#### 单个文件模式
- 直接保存到 `uploads/` 目录
- 返回文件路径

#### 压缩包模式
1. 解压到 `uploads/<timestamp>_<archive_name>/` 目录
2. 递归搜索题目文件（优先匹配名称包含 "problem"、"题目"、"question" 的文件）
3. 返回找到的题目文件的完整路径
4. 保留解压后的所有文件（数据、图片等）

### 3. 题目文件识别优先级

搜索策略：
1. **优先选择**：文件名包含以下关键词的 PDF/MD 文件
   - `problem`
   - `题目`
   - `question`
   - `题`
   
2. **备选方案**：压缩包中第一个找到的 PDF 或 MD 文件

3. **错误处理**：如果压缩包中未找到题目文件，返回 400 错误

### 4. 项目创建

找到题目文件后，系统将文件路径传递给 `launch_agents.sh new`，后续流程与单文件上传完全一致。

## 代码变更

### 后端 (`web/backend/app.py`)

```python
@app.post("/api/upload/problem")
async def upload_problem_file(...)
```

**主要增强**：
- 支持压缩包格式识别（通过文件扩展名）
- 使用 Python `zipfile` 和 `tarfile` 模块解压
- 递归搜索题目文件，支持关键词优先匹配
- 清理失败的解压操作
- 保留完整的解压目录供 Step 0 使用

### 前端 (`web/frontend/src/components/NewProjectModal.vue`)

**主要变更**：
1. 文件选择器 `accept` 属性增加压缩格式
2. 拖拽区域提示文本更新
3. 文件验证正则表达式扩展

```vue
<input ref="fi" type="file" 
  accept=".pdf,.md,.PDF,.MD,.zip,.tar,.gz,.bz2,.xz,.tgz,.tar.gz,.tar.bz2,.tar.xz"
```

## 使用示例

### 典型压缩包结构

```
cumcm2024_a.zip
├── 题目.pdf              # 题目文件（会被识别）
├── 附件1_数据.csv         # 数据文件
├── 附件2_图片.png         # 图片资源
└── 说明.txt              # 其他文档
```

### 上传流程

1. 用户在前端选择 `cumcm2024_a.zip`
2. 系统上传并解压到 `uploads/20260621_020530_cumcm2024_a/`
3. 识别到 `题目.pdf`，返回路径 `/path/to/uploads/20260621_020530_cumcm2024_a/题目.pdf`
4. 前端使用该路径调用 `/api/projects/new`
5. Step 0 agent 处理题目文件，可访问同目录下的所有数据文件

## Step 0 兼容性

Step 0 提示（`prompts/step0_problem_parsing.txt`）已支持处理绝对路径的 PDF/MD 文件：

- **PDF 路径**：调用 `mineru_parse.py` 解析
- **Markdown 路径**：直接拷贝到 `problem/source.md`
- **同目录文件**：Step 0 可以访问题目文件同目录下的所有数据文件和 sidecar 目录

因此，压缩包解压后的题目文件路径可以无缝传入现有流程。

## 错误处理

### 上传阶段
- **不支持的格式**：返回 400，提示支持的格式列表
- **文件过大**：返回 400，提示最大支持 100MB
- **解压失败**：返回 400，清理临时文件，报告具体错误

### 题目文件识别
- **未找到题目文件**：返回 400，提示"压缩包中未找到题目文件（PDF 或 Markdown）"
- 自动清理解压目录

## 注意事项

1. **解压目录命名**：`<timestamp>_<archive_stem>` 格式，避免冲突
2. **文件保留**：解压后的所有文件都会保留，供 Step 0 和后续步骤使用
3. **路径传递**：返回的是题目文件的完整绝对路径，不是解压目录路径
4. **安全性**：解压路径限制在 `uploads/` 目录下，防止路径遍历攻击
