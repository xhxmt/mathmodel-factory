## 改进后的咨询请求（新版）

```markdown
# 咨询请求：启动前 seed：题目解读与候选方法（贴 GPT Pro / Gemini Deep Think 的初步结论）

**元数据**
- gate: preflight
- step: 0
- project: cumcm2023A
- created: 2024-06-15 14:32:10

## 📋 项目背景

### 问题概要
```
# 2023年全国大学生数学建模竞赛A题

## 问题背景

某市计划建设一个多功能社区服务中心，需要在现有的20个候选地块中
选择若干地块建设，以最大限度地满足市民的需求。每个地块有不同的
成本、容量和覆盖范围。

## 数据说明

1. candidate_sites.csv - 候选地块数据
   - site_id: 地块编号
   - cost: 建设成本（万元）
   - capacity: 最大服务人数
   - location_x, location_y: 地理坐标

2. demand_points.csv - 需求点数据
   - point_id: 需求点编号
   - population: 人口数量
   - location_x, location_y: 地理坐标

3. distance_matrix.csv - 距离矩阵
   - 候选地块到需求点的距离（公里）

## 问题要求

1. 在预算5000万元内，选择地块使覆盖人口最大化
2. 每个需求点的服务距离不超过3公里
3. 给出地块选择方案及覆盖率
4. 分析预算与覆盖率的权衡关系
```

**当前所在步骤**: Step 0 — 问题解析与候选方法识别

## 🎯 决策影响

这是**启动前咨询**，你的决策将影响：
- Step 1 的背景研究方向（哪些领域、哪些前沿方法值得深入）
- Step 2 的建模提案范围（探索哪些技术路线）
- 整个项目的技术栈选择（优化器、求解器、工具链）

**建议关注点**：
- 问题的本质是什么？（优化/预测/分类/仿真/博弈…）
- 哪些前沿方法/论文值得参考？
- 有无特殊约束需要在建模初期就考虑？

## 📁 关键文件引用

以下是项目中已完成的关键文件，供你参考决策：

- `problem/problem_brief.md` — 问题解析
- `problem/constraints.md` — 约束条件分析
- `problem/data_inventory.md` — 数据清单

## 🤔 需要你（借助 GPT Pro / Gemini Deep Think）决定的事

请分析这个问题的本质特征，并建议初步的建模方向：

1. **问题分类**：这是一个什么类型的优化问题？
   - 设施选址问题（Facility Location Problem）
   - 覆盖问题（Coverage Problem）
   - 资源分配问题
   - 还是其他？

2. **核心挑战**：
   - 目标冲突：成本最小 vs 覆盖最大
   - 约束复杂度：预算、距离、容量
   - 数据规模：20个候选 × N个需求点

3. **候选建模方法**（请推荐2-3个）：
   - 经典方法：如 MILP、集合覆盖、P-中位问题
   - 启发式方法：如遗传算法、模拟退火
   - 现代方法：如多目标优化、鲁棒优化
   
   对每个方法，简述：
   - 适用性（为什么适合这个问题）
   - 求解难度（精确解 vs 近似解）
   - 实现难度（常见求解器支持情况）

4. **前沿参考**：
   - 是否有相关的 CUMCM/MCM 历年优秀论文？
   - 是否有近期的学术论文值得参考？
   - 是否有成熟的算法库/工具包可用？

## 💡 回答建议

请提供：
1. **明确的决策建议**（选哪个方案 / 采用什么策略）
2. **充分的理由**（为什么这样选？有何优势？）
3. **潜在风险提示**（这个选择可能遇到的坑）
4. **实施要点**（如果有具体的技术细节、参数建议、论文引用等）

你可以用 GPT Pro 的深度思考模式、或 Gemini 的 Deep Think 功能，来进行多角度分析。

## 🔄 回填方式

1. 把你的决策结论写进 `human_review.md` 的「## CONSULT preflight … STATUS: …」段落下；
2. 把该段落的 `STATUS: AWAITING` 改成 `STATUS: READY`；
3. 运行：`./launch_agents.sh resume cumcm2023A`

流水线会读取你的回填内容，并在后续步骤中优先参考。
```

**改进**：
- ✅ **项目背景完整**：30行问题摘要，一目了然
- ✅ **决策影响明确**：清楚地说明这个决策会影响后续哪些环节
- ✅ **关键文件可查**：列出已完成的参考资料
- ✅ **问题结构化**：把咨询拆分成 4 个子问题，逐一引导
- ✅ **回答框架清晰**：告诉人类应该提供什么信息
- ✅ **技术细节丰富**：提到了具体的方法名称、论文、工具

## 给顶级模型的价值

改进前，如果把简单的咨询请求发给 GPT Pro / Gemini Deep Think：

**输入**：
```
启动前 seed：题目解读与候选方法
```

**模型可能的问题**：
- ❓ 什么问题？没看到问题描述
- ❓ 候选方法是指什么？数学方法？编程方法？
- ❓ 要多详细？给3个还是10个？
- ❓ 有没有特殊约束？比赛规则？

**结果**：模型会给出泛泛的回答，或者反问一堆问题。

---

改进后，把完整的咨询请求发给 GPT Pro / Gemini Deep Think：

**输入**：
```
[完整的结构化咨询请求，包含背景、影响、文件、问题、建议]
```

**模型会得到**：
- ✅ 问题的完整描述（30行摘要）
- ✅ 数据结构（3个CSV文件的schema）
- ✅ 问题要求（4个子问题）
- ✅ 决策影响说明（这会影响后续哪些步骤）
- ✅ 回答框架（1-4点，每点的具体要求）

**结果**：模型能给出高质量、结构化、可直接使用的建议。

---

## 总结

| 维度 | 改进前 | 改进后 |
|------|--------|--------|
| 信息量 | 1句话 | 5个结构化区块 |
| 上下文 | 无 | 完整问题摘要 + 已完成文件列表 |
| 引导性 | 弱 | 强（4个子问题 + 回答框架） |
| 可操作性 | 低（不知道怎么回答） | 高（清楚地知道要提供什么） |
| UI体验 | 单一灰色文本框 | 分段式卡片 + emoji + 颜色高亮 |
| Markdown支持 | 纯文本 | 支持标题/列表/代码/粗体 |

**核心价值**：让顶级模型（GPT Pro / Gemini Deep Think）能够在**第一次提问**就获得**足够的上下文**，给出**高质量、可直接使用的建议**，而不是陷入多轮澄清问题的循环。
