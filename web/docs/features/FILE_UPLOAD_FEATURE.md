# Web 界面文件上传功能 - 使用指南

## 功能概述

用户现在可以直接通过浏览器上传题目文件（PDF 或 Markdown），无需手动将文件复制到服务器或输入文件路径。

## 新增功能

### 1. 双模式支持

在"新建项目"对话框中，用户可以选择两种方式提供题目文件：

#### 📤 上传文件（默认）
- 通过浏览器直接上传 PDF 或 Markdown 文件
- 支持点击上传和拖拽上传
- 实时上传进度显示
- 文件大小限制：100 MB

#### 📁 指定路径
- 传统方式：输入服务器上的文件路径
- 适合文件已经在服务器上的场景

### 2. 拖拽上传体验

**上传区域**：
```

```

**拖拽时高亮**：

**已上传文件显示**：

### 3. 文件验证

- ✅ **支持的格式**：`.pdf`, `.md`, `.PDF`, `.MD`
- ✅ **文件大小限制**：最大 100 MB
- ❌ 不支持的格式会提示错误

## 技术实现

### 前端 (`NewProjectModal.vue`)

#### 新增状态
```javascript
uploadMethod: 'file' | 'path'  // 上传方式切换
uploadedFile: File | null       // 已选择的文件
uploadProgress: 0-100           // 上传进度
isDragOver: boolean             // 拖拽状态
```

#### 核心功能
1. **文件选择**：点击触发 `<input type="file">`
2. **拖拽上传**：监听 `dragover`、`drop` 事件
3. **文件上传**：使用 `FormData` + `axios` 上传，监听 `onUploadProgress`
4. **路径获取**：上传成功后，后端返回服务器文件路径
5. **创建项目**：使用获取的路径创建项目

### 后端 (`web/backend/app.py`)

#### 新增端点
```python
POST /api/upload/problem
```

**功能**：
- 接收文件上传（`UploadFile`）
- 验证文件扩展名（`.pdf`, `.md`）
- 验证文件大小（≤100MB）
- 生成唯一文件名（带时间戳）
- 保存到 `uploads/` 目录
- 返回服务器文件路径

**返回格式**：
```json
{
  "status": "ok",
  "message": "文件上传成功",
  "file_path": "/home/tfisher/paper_factory/uploads/20240615_143022_2024_CUMCM_A.pdf",
  "filename": "20240615_143022_2024_CUMCM_A.pdf",
  "size": 2512345
}
```

#### 文件存储
- **目录**：`FACTORY_ROOT/uploads/`
- **命名规则**：`{timestamp}_{original_filename}`
- **时间戳格式**：`YYYYMMDD_HHMMSS`
- **示例**：`20240615_143022_2024_CUMCM_A.pdf`

#### 安全措施
1. **文件类型验证**：仅允许 PDF 和 Markdown
2. **文件大小限制**：100 MB 硬限制
3. **身份认证**：需要登录才能上传
4. **文件名清理**：添加时间戳前缀避免冲突

## 使用流程

### 方式 1：上传文件（推荐）

1. 点击 **"➕ 新建项目"** 按钮
2. 输入 **项目名称**（例如：`cumcm2024_a`）
3. 确保选择了 **"📤 上传文件"** 标签
4. **上传文件**（两种方式任选）：
   - **点击上传**：点击上传区域，选择文件
   - **拖拽上传**：将文件拖拽到上传区域
5. 等待文件上传完成（显示进度条）
6. 勾选选项：
   - ☐ 仅创建项目，不自动开始执行
   - ☐ 启用人工咨询模式
7. 点击 **"创建项目"** 按钮

### 方式 2：指定路径（传统）

1. 点击 **"➕ 新建项目"** 按钮
2. 输入 **项目名称**
3. 选择 **"📁 指定路径"** 标签
4. 输入 **服务器文件路径**（例如：`/home/user/problems/2024_A.pdf`）
5. 勾选选项
6. 点击 **"创建项目"** 按钮

## 示例场景

### 场景 1：本地有题目 PDF

**情况**：你的电脑上有 `2024_CUMCM_A.pdf`，服务器上没有

**操作**：
1. 打开 Web Dashboard
2. 新建项目 → 上传文件
3. 拖拽 PDF 到上传区域
4. 等待上传完成（2.4 MB / 100 MB）
5. 创建项目

**结果**：文件自动上传到服务器 `uploads/` 目录，项目启动

### 场景 2：服务器已有文件

**情况**：文件已经在服务器 `/data/problems/2024_A.pdf`

**操作**：
1. 新建项目 → 指定路径
2. 输入 `/data/problems/2024_A.pdf`
3. 创建项目

**结果**：直接使用服务器上的文件，无需上传

### 场景 3：从网盘下载后上传

**情况**：从比赛官网下载了题目

**操作**：
1. 下载 PDF 到本地
2. Web Dashboard → 新建项目
3. 上传刚下载的文件
4. 创建项目

**结果**：一站式完成文件传输和项目创建

## 配置说明

### 后端配置

在 `web/backend/app.py` 中调整上传限制：

```python
# 上传目录
UPLOAD_DIR = FACTORY_ROOT / "uploads"

# 最大文件大小：100MB
MAX_UPLOAD_SIZE = 100 * 1024 * 1024
```

### Nginx 配置（生产环境）

如果使用 Nginx 反向代理，需要增加上传大小限制：

```nginx
server {
    ...
    client_max_body_size 100M;
}
```

## 故障排除

### 问题 1：上传卡在 99%

**原因**：后端处理超时
**解决**：
```bash
# 检查后端日志
tail -f web/backend.log

# 重启后端
pkill -f "python.*app.py"
cd web && python3 backend/app.py &
```

### 问题 2：上传后找不到文件

**原因**：`uploads/` 目录不存在
**解决**：
```bash
mkdir -p uploads
chmod 755 uploads
```

### 问题 3：CORS 错误

**原因**：前端域名未加入白名单
**解决**：在 `app.py` 中添加域名到 `allow_origins`
